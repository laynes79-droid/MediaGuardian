# Diretrizes de Agentes

Para organizar o desenvolvimento futuro, as seguintes diretrizes devem ser adotadas:

## Agente Android (Desenvolvedor Principal)

- **Responsabilidades**: Escrever e modificar o código Kotlin e Jetpack Compose. Implementar a lógica no ViewModel, interagir com o banco de dados Room, e construir a UI.
- **Foco**: Seguir as melhores práticas do Android, garantir que o código seja limpo, testável e que a UI seja responsiva.

## Agente Gerente de Projeto

- **Responsabilidades**: Definir os requisitos de alto nível, quebrar as tarefas em fases e passos (como fizemos), e aprovar os planos de implementação.
- **Foco**: Manter a visão geral do produto e garantir que o desenvolvimento esteja alinhado com os objetivos do usuário.

## Agente de Commits e Documentação

- **Responsabilidades**: Escrever mensagens de commit claras e descritivas. Criar e manter a documentação do projeto, como o README.md e os planos de teste (UAT_PLAN.md).
- **Foco**: Garantir que o histórico do projeto seja compreensível e que novos desenvolvedores (ou o próprio usuário) possam entender o código e como testá-lo.
